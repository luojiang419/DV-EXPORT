# 部署文档

## 部署目标

本文档说明如何从本地源码构建达芬奇批量导出插件，并部署到 DaVinci Resolve Studio 的 Workflow Integration 插件目录。当前部署目标为 Windows。

## 环境要求

- Windows 10/11
- DaVinci Resolve Studio 19+
- Node.js
- Git
- GitHub CLI，可选，仅发布 GitHub Release 时需要
- 当前项目依赖已安装，或可执行 `npm install`

Resolve 插件安装目录：

```text
C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Workflow Integration Plugins\
```

`WorkflowIntegration.node` 常见来源：

```text
C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Developer\Workflow Integrations\Examples\SamplePlugin\WorkflowIntegration.node
```

## 本地验证

在构建前先执行基础检查：

```powershell
cd G:\data\app\DV-EXPORT\批量导出
npm run typecheck
npm test
```

预期结果：

- TypeScript 无类型错误
- Vitest 测试全部通过

## 构建插件

执行：

```powershell
cd G:\data\app\DV-EXPORT\批量导出
npm run build
```

构建脚本会完成：

- 使用 Vite 构建前端资源到 `build/web`
- 复制 `plugin-runtime`
- 合并前端资源和插件运行时
- 自动写入 `manifest.xml` 版本号
- 尝试复制 `WorkflowIntegration.node`
- 在 `dist/vX.Y.Z/com.dvexport.batch-export` 生成完整插件目录

如果产物中出现 `MISSING_WORKFLOWINTEGRATION_NODE.txt`，需要手动复制 `WorkflowIntegration.node` 到产物根目录。

## 本机安装

1. 关闭 DaVinci Resolve。
2. 备份当前已安装插件目录：

```powershell
$installed = 'C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Workflow Integration Plugins\com.dvexport.batch-export'
$backup = 'G:\data\app\DV-EXPORT\批量导出\backup\安装前已安装插件\com.dvexport.batch-export'
Copy-Item -LiteralPath $installed -Destination $backup -Recurse -Force
```

3. 复制新版本插件目录到 Resolve 插件目录：

```powershell
$source = 'G:\data\app\DV-EXPORT\批量导出\dist\v0.1.10\com.dvexport.batch-export'
$target = 'C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Workflow Integration Plugins\com.dvexport.batch-export'
Remove-Item -LiteralPath $target -Recurse -Force
Copy-Item -LiteralPath $source -Destination $target -Recurse -Force
```

4. 确认安装版本：

```powershell
Get-Content 'C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Workflow Integration Plugins\com.dvexport.batch-export\package.json'
```

5. 启动 DaVinci Resolve Studio，打开：

```text
Workspace -> Workflow Integrations -> 达芬奇批量导出
```

## 安装包制作

发布给用户时，建议压缩完整插件目录：

```powershell
$version = 'v0.1.10'
$pluginDir = "G:\data\app\DV-EXPORT\批量导出\dist\$version\com.dvexport.batch-export"
$package = "G:\data\app\DV-EXPORT\批量导出\dist\DV-EXPORT-$version-com.dvexport.batch-export.zip"
Compress-Archive -LiteralPath $pluginDir -DestinationPath $package -CompressionLevel Optimal
Get-FileHash -LiteralPath $package -Algorithm SHA256
```

压缩包结构应包含顶层 `com.dvexport.batch-export` 文件夹，解压后可直接复制到 Resolve 插件目录。

## 部署验收

安装后至少验证以下项目：

- 插件能在 Resolve 菜单中显示并打开
- 顶部环境信息能正确显示项目名、版本和产品
- 启动时出现“正在读取时间线信息...”加载动画
- 首次点击媒体池文件夹不再明显卡死
- 时间线列表能正确显示所选文件夹内的时间线
- 单选、Ctrl / Command 多选、Shift 连选正常
- 导出设置可以读取格式、编码器、分辨率和帧率
- 命名预览正常
- 不执行导出时，至少确认导出按钮启用条件正确


3. 将备份目录复制回安装目录：

## 常见问题

### 插件菜单中看不到入口

- 确认插件目录名是 `com.dvexport.batch-export`
- 确认目录内存在 `manifest.xml`、`main.js`、`preload.js`
- 确认安装位置是 `Workflow Integration Plugins`
- 重启 Resolve

### 打开插件提示初始化失败

- 确认使用的是 Resolve Studio，而不是免费版
- 确认 Resolve 主版本号为 19 或更高
- 确认插件根目录存在 `WorkflowIntegration.node`

### 导出设置读取失败

- 先在 Resolve Deliver 页面确认当前工程可用的格式和编码器
- 尝试切换格式后重新打开插件
- 查看插件返回的失败项，如格式、编码器、分辨率或帧率

### 首次打开仍有等待

`v0.1.10` 会在启动阶段预读取时间线信息。大项目仍可能需要等待 Resolve API 返回，但界面会显示加载动画，后续点击文件夹应复用缓存并更顺畅。
