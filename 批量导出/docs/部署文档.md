# 部署文档

## 部署目标

本文档说明如何从本地源码构建达芬奇批量导出插件，并部署到 DaVinci Resolve Studio 的 Workflow Integration 插件目录。当前部署目标为 Windows。

## 环境要求

- Windows 10/11
- DaVinci Resolve Studio 19+
- Node.js
- Git
- GitHub CLI，可选，仅发布 GitHub Release 时需要
- 当前项目依赖已安装，或可执行 `npm install`

Resolve 插件安装目录：

```text
C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Workflow Integration Plugins\
```

`WorkflowIntegration.node` 常见来源：

```text
C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Developer\Workflow Integrations\Examples\SamplePlugin\WorkflowIntegration.node
```

## 本地验证

在构建前先执行基础检查：

```powershell
cd G:\data\app\DV-EXPORT\批量导出
npm run typecheck
npm test
```

预期结果：

- TypeScript 无类型错误
- Vitest 测试全部通过

## 构建插件

执行：

```powershell
cd G:\data\app\DV-EXPORT\批量导出
npm run build
```

构建脚本会完成：

- 使用 Vite 构建前端资源到 `build/web`
- 复制 `plugin-runtime`
- 合并前端资源和插件运行时
- 自动写入 `manifest.xml` 版本号
- 尝试复制 `WorkflowIntegration.node`
- 在 `dist/vX.Y.Z/com.dvexport.batch-export` 生成完整插件目录

如果产物中出现 `MISSING_WORKFLOWINTEGRATION_NODE.txt`，需要手动复制 `WorkflowIntegration.node` 到产物根目录。

## 本机安装

1. 关闭 DaVinci Resolve。
2. 备份当前已安装插件目录：

```powershell
$installed = 'C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Workflow Integration Plugins\com.dvexport.batch-export'
$backup = 'G:\data\app\DV-EXPORT\批量导出\backup\安装前已安装插件\com.dvexport.batch-export'
Copy-Item -LiteralPath $installed -Destination $backup -Recurse -Force
```

3. 复制新版本插件目录到 Resolve 插件目录：

```powershell
$source = 'G:\data\app\DV-EXPORT\批量导出\dist\v0.1.18\com.dvexport.batch-export'
$target = 'C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Workflow Integration Plugins\com.dvexport.batch-export'
Remove-Item -LiteralPath $target -Recurse -Force
Copy-Item -LiteralPath $source -Destination $target -Recurse -Force
```

4. 确认安装版本：

```powershell
Get-Content 'C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Workflow Integration Plugins\com.dvexport.batch-export\package.json'
```

5. 启动 DaVinci Resolve Studio，打开：

```text
Workspace -> Workflow Integrations -> 达芬奇批量导出
```

## EXE 自安装包制作

发布给其他用户时，执行：

```powershell
cd G:\data\app\DV-EXPORT\批量导出
npm run build:installer
```

构建脚本会完成：

- 先执行现有插件构建流程，生成 `dist/vX.Y.Z/com.dvexport.batch-export`
- 校验产物中存在 `WorkflowIntegration.node`
- 生成 EXE 自安装包：`dist/installer/DV-EXPORT-vX.Y.Z-setup.exe`
- 生成 EXE 校验文件：`dist/installer/DV-EXPORT-vX.Y.Z-setup.sha256.txt`
- 生成 ZIP 备用安装包：`dist/installer/DV-EXPORT-vX.Y.Z-windows-installer.zip`
- 生成 ZIP 校验文件：`dist/installer/DV-EXPORT-vX.Y.Z-windows-installer.sha256.txt`

EXE 自安装包内部包含：

```text
payload.zip
install-windows.bat
install-windows.ps1
README.txt
```

分发给普通用户时，发送 `DV-EXPORT-vX.Y.Z-setup.exe` 和对应 `.sha256.txt` 即可。用户双击 EXE 后，安装器会自动：

- 请求管理员权限
- 检查 DaVinci Resolve 是否仍在运行
- 将插件安装到 Resolve Workflow Integration 插件目录
- 备份同名旧版本插件到 Resolve 插件目录下的 `backup` 文件夹
- 校验安装后的 `package.json` 版本

ZIP 备用包仍可用于排查问题：用户解压后双击 `install-windows.bat`，效果与 EXE 安装相同。

默认安装目录：

```text
C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\Workflow Integration Plugins\com.dvexport.batch-export
```

如果需要强制覆盖正在运行状态检测，可在解压目录中执行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\install-windows.ps1 -Force
```

EXE 构建依赖 Windows 自带的 `iexpress.exe`。如果构建机缺少该工具，需要换到完整 Windows 环境构建，或使用 ZIP 备用安装包。

## GitHub 发布

正式发布由 `.github/workflows/release.yml` 自动完成。推送到 `main` 后，流水线会根据源码版本下限与 Latest Release 计算下一个补丁版本，执行测试、云端构建、Draft Release、资产上传、远端复核和正式发布。

发布完成的必要条件：Actions 成功、Release 非草稿非预发布、tag 与正式分支指向同一提交、四个资产均为 `uploaded`，且远端 digest 与 SHA-256 文件一致。详见 [自动更新说明](./自动更新说明.md)。

以下命令仅用于查询，不再手工创建正式 Release：

前置检查：

```powershell
gh auth status
git status -sb
```

查询 Actions 与 Latest Release：

```powershell
gh run list --workflow release.yml --repo luojiang419/DV-EXPORT
gh api repos/luojiang419/DV-EXPORT/releases/latest
```

Release 说明建议包含：

- 版本号
- 主要变更
- 安装步骤
- SHA256 校验值
- 验证结果

## 部署验收

安装后至少验证以下项目：

- 插件能在 Resolve 菜单中显示并打开
- 顶部环境信息能正确显示项目名、版本和产品
- 启动时出现“正在读取时间线信息...”加载动画
- 首次点击媒体池文件夹不再明显卡死
- 时间线列表能正确显示所选文件夹内的时间线
- 单选、Ctrl / Command 多选、Shift 连选正常
- 导出设置可以读取格式、编码器、分辨率和帧率
- 帧率转换标签页可以显示目标帧率和新媒体夹名称
- 帧率转换能创建目标帧率的新时间线，并显示新时间线名称和目标媒体夹
- 转换成功后新时间线能在媒体池/时间线列表刷新后出现
- 命名预览正常
- 不执行导出时，至少确认导出按钮启用条件正确

## 常见问题

### 插件菜单中看不到入口

- 确认插件目录名是 `com.dvexport.batch-export`
- 确认目录内存在 `manifest.xml`、`main.js`、`preload.js`
- 确认安装位置是 `Workflow Integration Plugins`
- 重启 Resolve

### 打开插件提示初始化失败

- 确认使用的是 Resolve Studio，而不是免费版
- 确认 Resolve 主版本号为 19 或更高
- 确认插件根目录存在 `WorkflowIntegration.node`

### 导出设置读取失败

- 先在 Resolve Deliver 页面确认当前工程可用的格式和编码器
- 尝试切换格式后重新打开插件
- 查看插件返回的失败项，如格式、编码器、分辨率或帧率

### 首次打开仍有等待

`v0.1.18` 会在启动阶段预读取时间线信息，并支持在媒体池结构变化后手动刷新缓存。大项目仍可能需要等待 Resolve API 返回，但界面会显示加载动画，后续点击文件夹应复用缓存并更顺畅。批量导出按钮固定显示在插件首页右下角，右侧操作面板提供可编辑新帧率时间线创建；linked audio 或纯视频部分追加缺失时会尝试补追加并返回压缩诊断。
