# 自动更新说明

## 发布契约

- 仓库：`luojiang419/DV-EXPORT`
- API：`https://api.github.com/repos/luojiang419/DV-EXPORT/releases/latest`
- 标签：`vX.Y.Z`
- Windows 更新资产：`DV-EXPORT-vX.Y.Z-windows-installer.zip`
- 校验资产：`DV-EXPORT-vX.Y.Z-windows-installer.sha256.txt`
- 人工安装资产：`DV-EXPORT-vX.Y.Z-setup.exe`
- 版本解析：`max(package.json, Latest Release)` 后递增补丁版本

客户端只按标准化 tag 生成期望资产名并精确匹配。资产缺失、重复、大小为零、下载地址为空或 SHA-256 不一致时拒绝安装。

## 更新设置

更新策略和网络模式是两个独立维度。

更新策略：

- `自动更新`：默认；插件启动后检查并下载，完成后提示“立即更新 / 下次启动更新”。
- `手动更新`：启动时不请求网络，只在设置页点击“检查更新”后检测和下载。
- `禁止更新`：不检测、不下载、不显示提示，手动检查按钮不可用。

网络模式：

- `自动检测代理`：读取代理环境变量并探测 `127.0.0.1:7890`、`1080`、`8080`，未找到时直连。
- `手动代理`：仅使用通过校验的 HTTP、HTTPS、SOCKS4 或 SOCKS5 地址。
- `直连`：通过 curl `--noproxy *` 显式绕过代理。

默认值：

```text
updatePolicy=automatic
updateNetworkMode=automaticProxy
manualProxyUrl=http://127.0.0.1:7890
```

## 下载、延期与安装

更新状态保存在：

```text
%LOCALAPPDATA%\DV-EXPORT\updates
```

下载先写入 `.part`，完成后校验 GitHub asset digest 或配套 SHA-256、文件大小和精确资产名，再原子重命名为正式 ZIP。

- `立即更新`：启动外部 PowerShell 更新会话，确认进程创建后退出旧插件进程。
- `下次启动更新`：保留已验证的 pending；下次插件启动先复核，再启动一次更新会话。
- 外部更新器等待旧 PID 退出，提权运行安装脚本，成功或失败后按原命令重新打开插件。
- 安装器先在同盘 staging 完整校验，再移动旧目录到备份并原子替换；替换失败时恢复旧版本。

更新日志位于 `%LOCALAPPDATA%\DV-EXPORT\updates\logs`。网络失败、校验失败或 UAC 被拒绝不会阻止 Resolve 正常启动。

## 签名状态

`WorkflowIntegration.node` 使用 Blackmagic Design 的有效 Authenticode 签名，CI 会强制验证。当前 IExpress `setup.exe` 未配置公共代码签名，因此 Windows 可能显示 SmartScreen 或未知发布者提示；SHA-256 能验证完整性，但不能替代公共代码签名信誉。

## 自动发布

`.github/workflows/release.yml` 在 `main` push 或手动触发时执行：

```text
串行锁 → 计算版本 → 测试/类型检查 → 注入版本 → 构建安装包
→ 本地资产复核 → Draft Release → 上传 → 远端复核
→ 正式发布 → Latest/tag/提交/资产复核
```

同一提交已是 Latest 时幂等跳过。失败只清理由当前运行创建的 Release 和标签，不修改既有正式版本。
