# 已知限制

- Resolve 官方公开 API 对 Deliver 高级细项的覆盖并不完整，首期通过“基础预设 + 核心参数覆盖”的方式工作。
- 如果当前 Resolve 版本没有暴露 `Timeline.GetMediaPoolItem()`，同名时间线在极端情况下可能退化为按名称匹配。
- 本项目未清空既有 Render Queue；如果用户已有同名输出路径，Resolve 自身的覆盖策略仍由当前预设和软件设置决定。
- 帧率转换主入口当前创建可编辑的新时间线，不输出目标帧率视频文件。
- `模拟手动复制粘贴（实验）` 依赖 Windows、插件内置或用户配置的 AutoHotkey v2，以及 Resolve 主窗口焦点；执行期间用户不应操作键盘鼠标。
- FFmpeg/FFprobe 能力暂时保留在运行时和类型中，但不从帧率转换 UI 或主入口调用。
- 当前未实现子文件夹递归显示时间线。
