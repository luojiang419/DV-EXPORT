# 项目说明

## 项目概述

达芬奇批量导出插件是一个面向 `DaVinci Resolve Studio 19+` 的 Workflow Integration 插件，用于在 Resolve 内按媒体池文件夹定位时间线，并对选中的时间线批量创建导出任务。

项目当前以 Windows 环境为主要部署目标，插件运行在 Resolve 的 Workflow Integration 容器中，前端使用 React + TypeScript 构建，运行时通过 `WorkflowIntegration.node` 调用 Resolve API。

## 核心功能

- 读取当前 Resolve 工程信息，校验 Studio 版本与主版本号
- 展示媒体池文件夹树，按文件夹筛选直接包含的时间线
- 启动时预读取并缓存项目时间线信息，减少首次点击文件夹时的卡顿
- 时间线列表支持单选、Ctrl / Command 多选、Shift 连选
- 读取 Deliver 页面可用的预设、格式、编码器、分辨率和帧率选项
- 根据所选时间线帧率自动同步导出设置
- 持久化导出目录、命名模板和导出参数
- 批量为所选时间线创建 Render Queue 任务，并只启动本次创建的任务
- 保留用户已有 Render Queue，不执行清空队列操作
- 输出逐条导出结果，便于定位失败原因

## 技术栈

- UI 框架：React 18
- 语言：TypeScript
- 构建工具：Vite
- 测试框架：Vitest + jsdom
- 宿主环境：DaVinci Resolve Workflow Integration
- 桥接运行时：Electron preload + `WorkflowIntegration.node`
- 目标平台：Windows

## 运行机制

### 1. 插件启动

Resolve 加载 `plugin-runtime/main.js`，窗口页面加载打包后的前端资源。`plugin-runtime/preload.js` 通过 `contextBridge` 注入 `window.resolveBridge`，前端通过该 bridge 读取 Resolve 工程和执行导出操作。

### 2. 初始化数据

前端启动后并行读取：

- Resolve 环境信息
- 媒体池文件夹树
- Deliver 导出选项
- 项目时间线缓存

时间线缓存由 `prepareTimelineCache()` 主动预热。后续点击文件夹时，`getFolderTimelines(folderId)` 会复用缓存，只读取目标文件夹内的 clip 列表并匹配时间线记录。

### 3. 导出流程

用户选择文件夹和时间线后，前端根据导出配置生成导出计划，并调用 `runBatchExport()`。运行时逐条切换目标时间线、应用格式和编码器、设置 Render Settings、添加 Render Job，并启动本次创建的任务。

## 关键模块

- `src/app/App.tsx`：应用主状态、初始化流程、导出触发和页面布局
- `plugin-runtime/preload.js`：Resolve API 调用、缓存、导出执行和 bridge 暴露
- `src/core/exportPlanner.ts`：根据选中时间线和命名模板生成导出计划
- `src/core/renderSettings.ts`：前端导出配置到 Resolve Render Settings 的映射
- `src/core/exportSettingsStorage.ts`：导出设置本地持久化与兼容清理
- `src/core/timelineSelection.ts`：单选、多选、Shift 连选逻辑

## 版本与产物

源码包版本保存在 `package.json`，当前构建脚本会基于该版本自动寻找 `dist` 中下一个可用补丁版本目录。例如已有 `v0.1.9` 时，继续构建会生成 `v0.1.10`。

正式交付安装包建议通过 GitHub Release 发布，`dist` 目录只作为本机本地构建产物保存，不提交到 Git。

## 使用限制

- 当前只验证 Windows 安装路径
- 需要 DaVinci Resolve Studio 19 或更高版本
- 只操作当前打开工程
- 只展示所选媒体池文件夹直接包含的时间线
- 批量导出依赖 Resolve 当前可用的格式、编码器和分辨率组合
- `WorkflowIntegration.node` 必须存在于插件根目录

## 维护约定

- 功能修改前创建阶段备份到 `backup/`
- 每完成一个功能阶段，在 `进度快照/` 中新增递增编号快照
- 不删除历史 `dist` 版本目录
- 修改导出设置、时间线扫描、运行时 bridge 时，需要同时做类型检查、单元测试和实机验证
- GitHub Release 上传安装包，Git 仓库保留源码、文档、测试和项目记录
