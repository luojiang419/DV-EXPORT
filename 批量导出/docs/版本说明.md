# 版本说明

## v0.1.27

- 新增完整自动更新链路：启动检测、手动检查、精确 Release 资产选择、下载、SHA-256 校验和 pending 对账
- 设置页新增自动更新、手动更新、禁止更新三态，以及自动代理、手动代理、直连三种网络模式
- 下载完成后提供“立即更新 / 下次启动更新”，外部更新器等待旧插件进程退出后安装并重启
- 安装器改为 staging 校验、旧版本备份、原子替换和失败回滚
- 新增 GitHub Actions 自动补丁版本、Windows 云端构建、Draft Release、远端复核和正式发布
- `WorkflowIntegration.node` 在 CI 中强制验证 Blackmagic Design Authenticode 签名；IExpress 安装包当前仍未签名

## v0.1.26

- 将 `导出结果` 状态信息移动到底部固定操作栏左侧
- 移除独立的底部 `导出结果` 面板，减少底部空白区域
- 底部操作栏高度压缩为与主功能按钮一致的 `52px`
- 已重新构建并安装 Resolve 插件，安装目录版本同步为 `v0.1.26`

## v0.1.25

- 移除首页顶部常驻的 `环境检查通过` 成功提示条
- 保留错误提示和 Resolve 版本不兼容提示，避免异常状态静默
- 已重新构建并安装 Resolve 插件，安装目录版本同步为 `v0.1.25`

## v0.1.24

- 主进程改为单实例运行，重复从 Resolve 菜单打开时聚焦已有窗口，避免多个插件窗口堆叠
- 插件进程会监控启动它的父进程，Resolve 退出后自动退出，减少残留窗口被误认为随 DaVinci 自动启动的情况
- 时间线列表现在携带时间线分辨率，选择时间线后右侧导出设置会同步回填该时间线的分辨率和帧率
- 已重新构建并安装 Resolve 插件，安装目录版本同步为 `v0.1.24`

## v0.1.23

- 将帧率转换默认复制方式改为 `模拟手动复制粘贴（实验）`
- 前端下拉框默认选中模拟路线，后端未收到策略参数时也默认走 `resolve-ui-copy-paste`
- `内部 API 复制` 保留为手动回退选项
- 已重新构建并安装 Resolve 插件，安装目录版本同步为 `v0.1.23`

## v0.1.22

- 随插件打包官方 AutoHotkey v2 便携运行时 `AutoHotkey64.exe`
- `模拟手动复制粘贴（实验）` 不再要求用户先安装系统版 AutoHotkey；仍支持通过 `DV_EXPORT_AUTOHOTKEY_PATH` 覆盖运行时路径
- 已重新构建并安装 Resolve 插件，安装目录版本同步为 `v0.1.22`

## v0.1.21

- 新增帧率转换复制方式选项，默认仍使用现有内部 API 复制路线
- 新增实验路线 `模拟手动复制粘贴`：Resolve API 负责创建目标帧率时间线、切换源/目标时间线和校验，AutoHotkey v2 只负责向 Resolve 主窗口发送 `Ctrl+A/C/V`
- 主进程新增 AutoHotkey v2 路径检测和外部自动化 IPC；未安装或未配置时返回明确错误，不静默失败
- 混合路线失败时会删除已创建的半成品目标时间线，保留现有内部 API 路线作为回退选择

## v0.1.20

- 修复帧率转换 `11/12` 场景下，缺失定位仍使用过严签名误报多条并补错片段的问题
- 逐条追加现在记录每一次 `AppendToTimeline()` 的实际新增项目数，补追加优先针对真实 `actualAdded < expected` 的条目
- 失败诊断中真实未新增的条目显示为 `实际未新增`，避免把签名误差造成的多条 `疑似缺失` 当成真实缺口

## v0.1.19

- 修复帧率转换复制到目标帧率时间线时，仍沿用源时间线 `recordFrame` 导致片段落点被目标帧率重新解释的问题
- 复制计划会按源/目标时间线起始帧和帧率换算目标 `recordFrame`，降低同轨纯视频片段互相占位后只复制 `8/12` 的风险
- 主追加改为逐条追加，并以目标时间线实际新增音视频项目数为准，不再信任 Resolve `AppendToTimeline()` 的返回值
- 失败诊断会显示源帧到目标帧的映射以及 Resolve 返回数和实际新增数差异

## v0.1.18

- 修复纯视频时间线批量复制时，Resolve `AppendToTimeline()` 只落入部分视频项目导致 `7/12` 失败的问题
- 复制片段时按时间线实际时长裁剪过长的源入出点，降低同轨视频片段被前一个片段覆盖或挤掉的风险
- 主追加后若音视频项目数量不足，会按条补追加疑似缺失的视频或音频项目，不再只补 linked audio
- 缺失判断增加按媒体片段计数的宽松匹配，减少目标时间线已有项目被误报为缺失的情况
- 已构建并安装 Resolve 插件，安装目录版本同步为 `v0.1.18`

## v0.1.17

- 修复帧率转换复制 linked audio 场景下，Resolve all-media 追加后目标时间线少音频项目导致 `32/33` 失败的问题
- 复制计划保留 linked audio 元数据，主追加后若音视频项目数量不足，会定位疑似缺失音频并使用 `mediaType: 2` 补追加
- 复制失败原因增加源/目标每轨项目数、append 计划摘要、疑似缺失项目和补追加结果，便于定位无法复制的具体条目
- 保持帧率转换主方向为创建可编辑目标帧率时间线；FFmpeg/FFprobe 输出型能力仍不从 UI 或主入口调用
- 已构建并安装 Resolve 插件，安装目录版本同步为 `v0.1.17`

## v0.1.16

- 按用户确认的 C 路线，将帧率转换主流程恢复为创建可编辑的新帧率时间线
- `convertTimelineFrameRates()` 重新调用新时间线复制流程，不再要求输出目录、导出设置或转换策略
- 帧率转换 UI 恢复为目标时间线帧率、新媒体夹名称和新时间线结果展示
- 转换成功后自动刷新媒体池和时间线缓存，让新建时间线出现在列表中
- FFmpeg/FFprobe 运行时能力暂时保留，但不从 UI 或主入口调用
- 已构建并安装 Resolve 插件，安装目录版本同步为 `v0.1.16`

## v0.1.15

- 帧率转换路线改为目标帧率视频导出转换，不再创建可编辑的新时间线
- 新增 `resolve-render`、`ffmpeg-cfr`、`ffmpeg-motion` 三种转换策略
- 新增 FFmpeg/FFprobe 主进程能力，用于固定帧率转换、运动插帧转换和输出文件校验
- 帧率转换 UI 改为输出目录、转换策略和输出文件结果展示
- 转换结果会返回输出文件名、目标帧率校验、时长和音频流状态
- 已重新构建并安装 Resolve 插件，安装包版本同步为 `v0.1.15`

## v0.1.14

- 修复帧率转换复制已链接音视频剪辑时，目标时间线只落入部分项目导致 `4/8` 失败的问题
- 已链接的音视频项目改为按整体 clipInfo 追加，纯视频和纯音频项目仍按各自媒体类型追加
- 复制校验区分追加请求次数和目标音视频项目总数，失败提示更贴近实际原因
- 已重新构建并安装 Resolve 插件，安装包版本同步为 `v0.1.14`
