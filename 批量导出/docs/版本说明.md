# 版本说明

## v0.1.8

- 修复时间线选择后仍沿用默认 `24 fps` 导致 `SetRenderSettings` 被 Resolve 拒绝的问题
- 读取所选时间线的帧率并自动回填到导出设置面板
- 为时间线扫描增加加载遮罩和转圈提示，缓解点击文件夹时的卡顿感
- 导出目录、命名模板和导出参数改为本地持久化，重开插件后自动恢复

## v0.1.7

- 修复导出设置构造时将 `undefined` 可选字段一并传给 Resolve 的问题
- 将导出设置改为分步应用，并在失败时返回具体失败项，如分辨率、帧率或音频参数
- 提高批量导出失败时的可定位性，避免只显示通用的“应用导出设置失败”

## v0.1.6

- 压缩顶部标题栏高度、间距与字体，减少首屏占用
- 缩小底部导出结果栏的固定高度与内部字号，避免遮挡右侧导出设置和操作按钮
- 同步收紧状态栏与面板标题区间距，使主操作区可视空间更稳定

## v0.1.5

- 将插件页面改为固定视口布局，滚轮只在内容区内部滚动
- 侧边栏文件夹树改为按需展开，并缓存媒体池目录与时间线扫描结果以降低点击延迟
- 修复导出设置下拉菜单在暗黑主题下的浅底浅字问题
- 修复格式、编码器、分辨率联动读取，补充空状态提示并增强 Resolve 编码器取值兼容性

## v0.1.4

- 修正 `manifest.xml` 为 Resolve 示例兼容格式
- 新增 `FilePath` 并补齐 `BlackmagicDesign` 根节点
- 修复构建脚本在 Windows Node 环境下无法自动复制 `WorkflowIntegration.node` 的问题
- 产物现在可自包含安装到 Workflow Integration 插件目录

## v0.1.3

- 首次修正插件描述文件格式并重装到 Resolve 插件目录

## v0.1.2

- 将界面整体改为达芬奇专业深灰暗黑模式
- 更新插件窗口底色，避免启动时出现浅色闪屏
- 重做全局色板、卡片、列表、表单和按钮的深灰视觉体系
- 保留橙色高亮作为 Resolve 风格的重点操作色

## v0.1.1

- 初始化 Workflow Integration 插件项目结构
- 实现 Resolve 环境校验与 bridge
- 实现媒体池文件夹树与时间线列表
- 实现 Ctrl / Shift 多选
- 实现导出设置面板与命名模板
- 实现批量创建 render job 并启动本次任务
- 补充单测、安装文档、已知限制说明

## 版本目录

- `dist/v0.1.0`: 首次构建产物
- `dist/v0.1.1`: 修正类型与前端逻辑后的构建产物
- `dist/v0.1.2`: 暗黑主题改造后的构建产物
- `dist/v0.1.3`: 首次修正插件描述文件格式后的构建产物
- `dist/v0.1.4`: 修复自动打包 `WorkflowIntegration.node` 后的最新构建产物
- `dist/v0.1.5`: 修复固定布局、侧边栏性能和导出选项联动后的最新构建产物
- `dist/v0.1.6`: 压缩标题栏和导出结果栏布局后的最新构建产物
- `dist/v0.1.7`: 修复导出设置应用失败并细化失败原因后的最新构建产物
- `dist/v0.1.8`: 修复时间线帧率同步、扫描加载态和导出设置持久化后的最新构建产物
