# 测试清单

## 已自动验证

- 命名模板 token 展开
- Windows 文件名非法字符清洗
- Render Settings 映射
- 批量导出任务计划生成
- Ctrl / Shift 多选逻辑
- 帧率转换时间线命名、目标媒体夹规范化、输出型辅助能力和结果汇总
- 帧率转换会按源/目标起始帧和帧率换算目标时间线 `recordFrame`
- 帧率转换 linked audio 缺失时应返回源/目标轨道计数、append 计划摘要和疑似缺失条目
- 帧率转换纯视频部分追加失败时应尝试按条补追加，并在失败时返回补追加项目统计
- 帧率转换逐条追加返回成功但实际未新增时，应优先使用真实未新增条目补追加并显示 `实际未新增`
- 帧率转换策略归一化支持 `resolve-ui-copy-paste`
- 帧率转换未传策略或策略未知时默认走 `resolve-ui-copy-paste`
- 选择时间线后，导出设置可同步该时间线的分辨率和帧率
- TypeScript 类型检查
- Vite 生产构建
- 更新策略与网络模式 3×3 九种组合
- 版本高于、等于、低于以及 `v` 前缀比较
- Release 多资产精确选择、缺失、重复和 SHA-256 文件解析
- pending 版本、资产名、大小、路径和摘要契约
- 自动发布版本解析器边界与幂等规则

## 建议在 Resolve 中手动验证

- 启动插件后能正确识别 Resolve Studio 19+
- 能展示媒体池文件夹树
- 选择某个文件夹后，只显示该文件夹内的时间线
- Ctrl 多选、Shift 连选行为符合预期
- 首页顶部不再显示常驻的 `环境检查通过` 成功提示条
- `导出结果` 状态显示在底部操作栏左侧，底部栏高度与主功能按钮一致
- 选择不同分辨率或帧率的时间线后，右侧 `导出设置` 面板同步显示该时间线的分辨率和帧率
- 切换格式后编码器列表联动更新
- 关闭 Resolve 后插件窗口/进程不应继续残留；再次从 `Workspace -> Workflow Integrations -> 达芬奇批量导出` 手动打开时能正常显示
- 右侧 `帧率转换` 标签页能选择目标帧率和新媒体夹名称
- 右侧 `帧率转换` 标签页默认选择 `模拟手动复制粘贴（实验）`，并保留 `内部 API 复制` 回退选项
- 转换后能创建目标帧率的新时间线，并显示新时间线名称和目标媒体夹
- 选择 `模拟手动复制粘贴（实验）` 时，应优先使用插件内置 AutoHotkey v2 便携运行时通过 Resolve 主窗口完成复制粘贴
- 源时间线与目标时间线帧率不同时，连续视频片段在目标时间线中应按时间位置换算后完整落入，不应只复制部分片段
- 若 Resolve 返回追加成功但目标时间线项目数未增加，失败日志应只列出真实未新增条目
- 含已链接多轨音频的时间线转换后，目标时间线音视频项目数量应完整；若 Resolve 拒绝某条音频，应显示疑似缺失条目
- 纯视频多片段时间线转换后，目标时间线视频项目数量应完整；若 Resolve 拒绝某条视频，应显示疑似缺失条目和补追加项目统计
- 新时间线能在媒体池/时间线列表刷新后出现
- 转换失败时能返回逐条失败原因
- 选择输出目录后能够创建多条 render job
- 批量导出时不会清空原有 Render Queue
- 导出失败时能返回逐条结果说明
- 设置页可切换三种更新策略和三种网络模式
- 自动更新完成后只显示一个全局提示，两个按钮语义正确
- 真实旧版本可从 GitHub Latest Release 下载、安装并显示新版本

## 特殊场景

- 空文件夹
- 同名时间线
- 未选择输出目录直接点击导出
- Resolve 已在渲染中时点击导出
- 预设加载失败
- 源时间线已是目标帧率
- 含调色、字幕、Fusion、多轨音频的复杂时间线帧率转换
