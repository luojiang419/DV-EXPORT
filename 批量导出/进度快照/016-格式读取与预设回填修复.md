# 016 格式读取与预设回填修复

## 已完成内容

- 定位到多种格式切换后编码器与分辨率读取失败的根因，是 Resolve 导出格式存在“显示名/内部 token”不一致
- 调整格式读取逻辑，查询编码器、分辨率和设置导出格式时都会同时尝试候选格式 token
- 修复基础预设会被自动回填为 `H.264 Master` 的问题
- 为导出设置持久化补充“预设是否为手动选择”的元数据，兼容清理旧版本误写入的默认预设
- 补充持久化相关测试，完整测试与类型检查通过
